### Основы разработки ИИ-агентов: Комплексное учебное пособие

Данное пособие составлено на основе пятидневного курса по изучению ИИ-агентов и охватывает темы от базовой архитектуры до вывода систем в промышленную эксплуатацию.

#### Часть 1: Проверочный тест (Quiz)

Ответьте на следующие вопросы, опираясь на материалы курса. Каждый ответ должен состоять из 2–3 предложений.

1. **В чем заключается принципиальное различие между обычной языковой моделью (LLM) и ИИ-агентом?**  
2. **Какие три ключевые способности определяют полноценного ИИ-агента?**  
3. **Перечислите основные компоненты архитектуры Google Agent Development Kit (ADK).**  
4. **Какую роль играют docstring-строки и аннотации типов при создании инструментов для агента?**  
5. **В чем разница между использованием инструментов (Agent Tools) и использованием субагентов (Sub-Agents)?**  
6. **Что такое протокол MCP и какую основную проблему он решает?**  
7. **Разграничьте понятия «Сессия» и «Устойчивая память» в контексте ADK.**  
8. **Почему традиционные подходы к тестированию (QA) плохо работают с ИИ-агентами?**  
9. **Что подразумевает концепция «LLM-as-a-Judge»?**  
10. **Что такое Agent Card и для чего она используется в протоколе A2A?**

#### Часть 2: Ответы на тест (Answer Key)

1. **Ответ:**  В то время как LLM работает по пассивной схеме «промпт → текст», агент использует цикл «размышление → действие → наблюдение». Агент способен самостоятельно определять шаги для решения задачи, тогда как обычная модель требует участия человека на каждом этапе.  
2. **Ответ:**  Агент должен уметь воспринимать информацию из окружающей среды, планировать последовательность действий для достижения цели и действовать автономно с помощью инструментов. Эти функции превращают изолированную модель в активное приложение.  
3. **Ответ:**  Архитектура ADK включает четыре ключевых элемента:  *Agent*  (определение модели и инструкций),  *Runner*  (оркестратор диалога),  *Tools*  (внешние инструменты) и  *Session*  (контекст диалога). Этот фреймворк является модульным и оптимизирован для работы с моделями Gemini.  
4. **Ответ:**  LLM использует docstring-строки как инструкции, чтобы понять, в какой ситуации и как именно вызывать конкретную функцию. Аннотации типов позволяют ADK автоматически генерировать корректную JSON-схему, необходимую для взаимодействия модели с программным кодом.  
5. **Ответ:**  При делегировании (Tools) агент вызывает другой компонент для выполнения расчета и получает результат, оставаясь главным в цикле. При передаче управления (Sub-Agents) основной агент выходит из цикла, полностью передавая контроль другому агенту, например, при эскалации задачи в службу поддержки.  
6. **Ответ:**  MCP (Model Context Protocol) — это открытый стандарт для подключения ИИ к внешним данным и инструментам через единый интерфейс. Он обеспечивает интероперабельность между разными моделями и позволяет безопасно подключать файловые системы или базы данных.  
7. **Ответ:**  Сессия представляет собой кратковременную память, хранящую историю сообщений в рамках одного конкретного разговора. Устойчивая память (Persistent Memory) — это долгосрочное обучение, позволяющее агенту запоминать предпочтения пользователя и переносить знания между разными сессиями.  
8. **Ответ:**  Главная сложность заключается в недетерминированности агентов: один и тот же запрос может приводить к разным результатам при разных запусках. Это требует перехода от проверки фиксированных ответов к комплексной оценке траектории рассуждений и качества использования инструментов.  
9. **Ответ:**  Эта концепция предполагает использование более мощной языковой модели в качестве «судьи» для оценки ответов разрабатываемого агента. Такой подход масштабируется на тысячи тестов и обходится дешевле и быстрее, чем ручная проверка человеком.  
10. **Ответ:**  Agent Card — это своего рода «визитная карточка» агента, содержащая описание его возможностей. В протоколе Agent-to-Agent (A2A) она необходима для того, чтобы другие системы могли понять, какие задачи можно поручить данному агенту.

#### Часть 4: Глоссарий ключевых терминов

Термин,Определение  
ADK (Agent Development Kit),"Open-source фреймворк от Google для создания ИИ-агентов на Python, Java и Go."  
Agent2Agent (A2A),Протокол для стандартизированного взаимодействия и обмена задачами между агентами из разных систем.  
Context Engineering,Процесс динамической сборки и управления информацией внутри контекстного окна LLM.  
Guardrails,"Механизмы-ограничители, предназначенные для предотвращения нежелательного или опасного поведения агента в реальной среде."  
LLM-as-a-Judge,"Метод оценки качества работы агента, при котором результаты его работы проверяются другой (часто более мощной) языковой моделью."  
MCP (Model Context Protocol),Открытый стандарт взаимодействия между ИИ-клиентами и серверами данных/инструментов.  
Observability (Наблюдаемость),"Комплексный мониторинг системы через логи (события), трейсы (путь запроса) и метрики (производительность и стоимость)."  
Runner,"Компонент ADK, выполняющий роль оркестратора, который управляет ходом диалога и вызовами инструментов."  
Session,"Контейнер для диалога, который хранит историю сообщений и поддерживает контекст конкретного разговора."  
ToolContext,"Механизм, позволяющий инструментам считывать или записывать данные непосредственно в состояние текущей сессии агента."
